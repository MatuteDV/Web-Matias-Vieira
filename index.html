Habilidades

HTML5 Jr.

Si hablamos de HTML5 no solo hablamos de maquetación web. Hablamos de un estándar, etiquetas nuevas, maquetado semántico, estilos y efectos con CSS, comportamiento con JavaScript y dinamismo con Ajax.

Pero también tenemos que hablar de ECMAScript, DOM, BOM, eventos, JSON, Cookies y APIs (Workers, Plug-ins, Web Storage, etc). Podemos entonces agrupar dentro de “saber” HTML5 a saber DHTML, XHTML y SPA.

CSS3 Jr.

HTML5 busca, entre otras cuestiones, mejorar la legibilidad del código de maquetación; delegando a CSS todo el estilizado, que antes se implementaban mediante atributos de etiquetas. Claro que CSS ya es mucho más potente que eso desde antes de CSS3 y por esto para hablar de este lenguaje de estilizado, podemos resaltar:

Variables.
Selectores nuevos por clase, pseudo-clases y pseudo-elementos, atributos y combinando estos.
Edición de imágenes, tanto forma como efectos de imagen.
Nuevas, mejores y practicos Layout: Flexbox, Grid View y Grid Container.
Y otras cuantas a destacar: Animaciones, contadores, sombras, transforms, rounded corners, SVG y Canvas. Entre otros más claro.

JavaScript / JQuery / AngularJs - (Jr.)

Librerías como JQuery y Frameworks Js como AngularJS, son hoy en día herramientas claves para desarrollar de manera estandarizada y más legible, codificar más rápido con funciones testeadas y mejoras de seguridad; abstrayendo operaciones complejas que podemos invocar .

Cada Framework tiene sus ventajas y por esto conocer que puede hacer, compatibilidad y como trabaja cada Framework es importante.

Uno problema de su implementación es el peso agregado de carga de la librería/Framework, su compatibilidad con los distintos navegadores y a veces la necesidad de modificar código por actualizaciones de estos. Aquí es donde, si no es totalmente necesario/beneficioso un Framework, es importante conocer muy bien JavaScript Vanilla (JS puro).

Seguir leyendo

Cloudinary - WebP

La optimización es un factor importante para una web. Una web, además que debe ser responsiva, es necesario que no demore en cargar minimizando que los usuarios abandonen la web por esperas de carga. Y como una Web tiene mucho contenido de imágenes y videos, es importante su tamaño, formato, calidad y compresión. Pero además que el servidor tenga buenos tiempos de respuesta para brindarlos fluidamente.

Es óptimo tercerizar la carga de imágenes y videos a un hosting dedicado, como lo es Cloudinary, dado que quitaremos tráfico de nuestro servidor, aumentando los tiempos de respuesta de nuestro servidor y optimizando la entrega "multimedia" dinámicamente. Debido a que podremos:
Solicitar las imágenes y videos adaptados dinámicamente su formato (Webp, Jpg, etc.), calidad, compresión, forma y tamaño al dispositivo desde el que se accedió.
Distribuir el contenido en la red a fin que haya una copia próxima a cada cliente que accede.

Responsive Design Mobile First

Hace ya tiempo que un sitio web debe ser responsivo para existir. Los motores de búsqueda priorizan estos sitios, como las buenas practicas y los sitios semánticos, y sumado al alto porcentaje de sitios visitados desde dispositivos móviles; se hace imprescindible desarrollar una página pensándola primero para móvil y adaptable para pantallas de escritorio. Que además hace más rápida su carga.

Para esto hay que conocer y saber aplicar MediaQuerys, medidas porcentuales (%, vw, vh, em, rem, etc.), entre otras propiedades y atributos, como también manejar Frameworks Front-end como Bootstrap o algunos mas completos o livianos como W3css, que nos ayudan con estas prácticas antes mencionadas y nos brindan herramientas/mejoras estéticas.

Back-End: NodeJS, PHP y SQL - (Jr.)

PHP es un lenguaje muy practico y ligero para casi cualquier proyecto y con diferentes métodos de implementación para conectar bases de datos estructurales SQL. NodeJS es JavaScript del lado del servidor, pensado en ejecución asincrona no-bloqueante para uso intensivo de datos en tiempo real. Su implementación es muy común haga junto a MongoDB, base de datos no-SQL.

También a futuro no lejano quiero comenzar a desarrollar como MEAN y MERN Stack. MEAN Stack es un Full-Stack que utiliza las tecnologías MongoDB, ExpressJS, AngularJS y NodeJS, y MERN utiliza ReactJS en lugar de Angular. Desarrollando plenamente en JavaScript pero donde el mayor cambio esta en la base de datos con No-SQL; que es muy práctico en conjunto a JSON y rápido con grandes volúmenes de datos.

Donde encontrarme?

Nací, crecí, estudié y vivo actualmente en la ciudad de Paraná, Entre Rios, Argentina (CP: 3100). Capital de la provincia de Entre Ríos, se encuentra ubicada sobre la margen izquierda del río que le da su nombre y que en lengua guaraní significa “pariente del mar”. Es poseedora de una vista incomparable del río y sus islas, y atardeceres de una belleza sin igual, con sus singulares y bellas barrancas de mas de 80 metros de altura del parque Urquiza.

Entre 1853 y 1861 fue la capital de la Confederación Argentina. Paraná fue formándose en la época de la colonización española, pero no fue fundada como ciudad, sino que su poblamiento fue gradual.

Entre sus atractivos característicos prevalecen ineludibles la renovada Avenida Costanera, el siempre encantador Parque Urquiza, el Puerto en sus distintas acepciones, la imponente Catedral y el solemne Teatro 3 de Febrero.
